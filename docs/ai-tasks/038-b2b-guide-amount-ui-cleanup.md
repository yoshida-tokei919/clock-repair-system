# Task 038: B2B共有ページ ご案内金額UI整理とホイールピッカー化

## 目的

B2B共有ページの「ご案内金額」周辺UIを整理し、スマホで見やすく、実務で使いやすい計算補助にする。

現在、機能的にはご案内金額入力や出力連動が進んでいるが、画面上では以下の課題がある。

- ご案内金額欄がやや詰まって見える
- 税込合計、見積内容、ご案内金額、説明文、メモの情報強度が近く、画面がガチャついて見える
- 案件情報が重複している
- 見積内容 / PDF / 説明文 / メモなど補助情報が主役に近く見える
- 倍率選択が通常selectで、スマホ操作としてやや味気ない

今回の目的は、新機能追加というより、既存のご案内金額機能をスマホB2B業務に合うUIへ整理すること。

## 背景

B2B先にとって、この共有ページの価値は単なる見積確認ではなく、案件ごとに以下を完結できること。

- 当店見積額を確認する
- エンドユーザーへのご案内金額を決める
- 必要なら説明文をコピーする
- 承認 / 差戻しする
- コメントする
- 自社内用メモを残す、または外部メモアプリへ共有する

その中でも、ご案内金額の計算補助はB2B先にとって便利な機能なので、見やすく整理する。

## 絶対方針

- 既存コードを壊さず最小差分
- DB保存しない
- API追加しない
- 当店側には送信しない
- コメント/承認/差戻し/アコーディオンを壊さない
- CSV/メモ出力との連動を壊さない
- B2C共有ページに影響させない
- `.claude/settings.local.json` は触らない
- `.next-dev.err.log` は触らない
- 本番DB/Supabase/Railway DBは触らない
- `git add .` は使わない

## UI禁止ワード

画面表示、CSV、メモ用テキストに以下を出さない。

- 掛け率
- 掛率
- マークアップ率
- 利益
- 利益額
- 利益率
- 粗利
- マージン
- 差額

## 使用してよい表現

- 当店見積額
- ご案内金額
- +¥15,000
- ×1.50 相当
- 目安
- 計算補助

※ 内部実装変数名は `markupRate` などでもよいが、UI文言には出さない。

## 修正内容

### 1. ご案内金額ブロックを縦積みに整理する

現在のご案内金額欄は、入力欄・倍率select・補助表示・説明文が詰まり気味。

スマホでは横並びを避け、以下のように縦積みにする。

```text
ご案内金額

[ ¥30,000 ]

+¥15,000
×2.00 相当

目安
[ホイールピッカー]
```

重要:

- 入力欄と倍率選択を横並びにしない
- 補助表示は入力欄の下
- `+¥...` と `×... 相当` は小さなチップでよい
- ただし読めないほど小さくしない
- 「差額」というラベルは表示しない

### 2. 倍率選択をホイールピッカー風UIにする

スマホ前提なので、通常selectではなく、ホイールピッカー風UIに変更する。

範囲:

- ×1.0〜×3.0
- 0.1刻み

表示例:

```text
目安
×1.0
×1.1
×1.2
...
×3.0
```

UI方針:

- 本格的なネイティブピッカーでなくてもよい
- MVPでは縦スクロール式の選択UIでよい
- 選択中の倍率を中央または強調表示する
- `×1.50 相当` は小数第2位で表示
- ラベルに「掛け率」は使わない
- 見出しは `目安` または `計算補助` 程度にする

実装案:

- 案A: 縦スクロールリスト
- `overflow-y-auto`
- 高さ固定
- 倍率候補を縦に並べる
- 選択中だけ青枠/青背景
- タップで選択
- 案B: compact picker
- 左右または上下ボタンで倍率を変更
- 表示は `×1.5`

ただし、ヨシダさんの希望はホイールピッカー寄りなので、案A優先。

### 3. 相互連動

倍率を選択した場合:

当店見積額 ¥15,000 で ×2.0 を選択すると、

```text
ご案内金額: ¥30,000
+¥15,000
×2.00 相当
```

ご案内金額を手入力した場合:

当店見積額 ¥15,000 で ¥25,000 と入力すると、

```text
+¥10,000
×1.67 相当
```

+金額の手入力:

今回は実装しない。`+¥...` は表示専用でよい。

理由:

状態管理が複雑になり、今回のUI整理Taskの範囲を超えるため。

### 4. 表示順を整理する

展開中案件カードの主役は以下。

- 税込合計
- ご案内金額
- 承認 / 差戻し

補助情報は一段弱く見せる。

推奨順:

1. 案件の最小情報
2. 税込合計
3. ご案内金額ブロック
4. 承認 / 差戻し
5. 見積内容
6. PDFを見る
7. エンドユーザー様への説明文
8. コメント
9. 貴社専用メモ

理由:

B2B担当者の思考順は、

```text
どの案件？
いくら？
いくらで案内する？
承認する？
必要なら詳細確認・コメント・メモ
```

だから。

### 5. 展開後ヘッダーの重複を減らす

閉じたアコーディオン見出しにすでに案件概要があるため、展開後ヘッダーは簡潔にする。

表示例:

```text
Y-008
ROLEX デイトジャスト
Ref: 16233
未確認
```

顧客名・ブランド・モデル・Ref・ステータスを何度も強く出しすぎない。

### 6. 見積内容とPDFの見た目を一段弱める

見積内容は必要だが、主役ではない。

- 明細は読みやすく
- ただし装飾は控えめ
- PDFボタンは黒ベタで強すぎる場合、青枠または淡いボタンへ変更してよい

### 7. 貴社専用メモを少し控えめにする

貴社専用メモは補助機能。

現在大きく目立つ場合は、以下のように調整する。

- textarea高さを少し抑える
- 初期表示はコンパクトにする
- ただし入力しにくくしすぎない
- 薄い黄色/クリーム系は維持

文言:

```text
貴社専用メモ
この内容は当店には送信されません。
```

### 8. CSV/メモ出力との連動維持

Task037で追加した以下を壊さない。

- CSVコピー
- メモ用テキストコピー
- メモアプリ共有
- 出力項目選択
- ご案内金額の出力

ご案内金額が倍率選択で自動入力された場合も、CSV/メモ用テキストに反映する。

未入力なら空欄出力でよい。
