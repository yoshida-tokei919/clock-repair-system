# AI Task 003: 見積・修理明細欄に税抜価格表示を追加する

## 目的

見積・修理明細欄で、各明細行の金額について「税抜価格」が分かるように表示する。

実務上、明細行は税抜価格で確認でき、合計欄で消費税と税込合計を確認できる形が自然。
そのため、修理明細・見積明細の表示上、税抜金額を明示する。

## ブランチ

feature/show-tax-excluded-price-in-line-items

## 触ってよい範囲

- `src/components/repairs/RepairEntryForm.tsx`

## 触ってはいけない範囲

- `prisma/schema.prisma`
- `src/app/api/repairs/route.ts`
- `src/app/api/repairs/[id]/route.ts`
- `src/actions/document-actions.ts`
- `/api/*`
- DB schema
- 保存ロジック
- 見積計算ロジック本体
- 帳票生成ロジック
- UI全体の大改修

## 前提

- 今回は「表示」の改善だけを行う。
- DB保存値は変更しない。
- 税計算ロジックは変更しない。
- 合計金額、消費税、税込合計の計算式は変更しない。
- `RepairEntryForm.tsx` は巨大で壊れやすいため、修正範囲を見積・修理明細欄の表示周辺に限定する。
- 不要なリファクタリングは禁止。
- unrelated file は変更しない。

## 実装要件

- 見積・修理明細欄の各明細行で、税抜価格が分かる表示を追加する。
- 既存の金額入力・表示を壊さない。
- 既存の数量、単価、小計、合計欄の計算挙動を変更しない。
- 技術料行・交換部品行の両方で税抜価格が確認できるようにする。
- 保存時の payload は変更しない。
- API routes は変更しない。
- Prisma schema は変更しない。
- 帳票生成処理は変更しない。
- 税率や丸め処理を新しく作り直さない。
- 既存コードに税抜金額または小計表示の変数がある場合は、それを優先して使う。
- 既存の金額がすでに税抜扱いなら、表示ラベルとして「税抜」を付けるだけでよい。
- 既存の金額が税込扱いの場合は、既存の税率・計算ロジックを調査してから最小差分で表示する。ただし計算本体は変更しない。

## 重要な確認ポイント

まず実装前に、`RepairEntryForm.tsx` 内で以下を確認すること。

- 明細行の金額が税抜として扱われているのか
- 明細行の金額が税込として扱われているのか
- `taxAmount`
- `totalAmount`
- `partsTotal`
- `technicalFee`
- `quantity`
- `price`
- `unitPrice`
- `amount`
- `subtotal`

上記のうち、実際に使われている変数・プロパティを確認する。

## 想定される修正方針

### パターンA：既存の明細行金額が税抜扱いの場合

この場合は、計算処理を変更せず、表示ラベルだけ追加する。

例：

<span>
  ¥{amount.toLocaleString()}
</span>
