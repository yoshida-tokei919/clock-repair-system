# AI Task 017: 右部品パネルの役割とUI設計

## 目的

右部品パネルの役割を再定義し、今後のUI・Web検索・PartsMaster登録・明細挿入導線を整理する。

今回は設計のみとし、実装は行わない。

## 重要方針

- 明細欄では検索条件を絞る。
- 右部品パネルでは条件に合うPartsMaster候補を表示し、候補選択・追加絞り込み・Web検索・新規登録・明細挿入を行う。
- Web検索UIはダイアログではなく、右部品パネル内に常時表示する。
- 検索結果メモ/詳細エリアは作らない。
- Web検索結果や在庫部品情報は、新規登録 / 在庫部品登録フォームへ直接入力する。
- 画像生成したUIをそのまま再現しない。
- 既存UIをベースに、今後小Taskで段階的に改善する。

## 役割分担

### 明細欄

明細欄は、見積・修理明細そのものを編集する場所とする。

明細欄では以下を行う。

- 技術料 / 交換部品の区分選択
- 部品種別、カテゴリ、部品名などの検索条件指定
- 対象明細行から右部品パネルを開く
- PartsMaster選択後の反映結果確認

明細欄に過度な検索UIや登録UIを増やさない。

### 右部品パネル

右部品パネルは、明細行に紐づく部品探索と反映の作業場所とする。

右部品パネルでは以下を行う。

- 対象明細行の検索条件を受け取る
- 条件に合うPartsMaster候補を表示する
- キーワード、部品種別、Cal、Refなどで追加絞り込みする
- Web検索先を選択し、検索を実行する
- PartsMaster候補を対象明細行へ反映する
- 候補がない場合、新規PartsMaster登録 / 在庫部品登録へ進む
- 登録後、明細行へ挿入または反映する

## Web検索UI

Web検索UIは、現在の検索ダイアログから右部品パネル内へ移す方針とする。

右部品パネル内に常時表示する要素は以下。

- 検索語プレビュー
- 検索サイト一覧
- 検索実行ボタン
- サイト追加 / 削除などの最小限の管理導線

Web検索はPartsMaster候補表示と同じ作業文脈に置き、ユーザーがダイアログを往復しなくて済む状態を目指す。

## PartsMaster候補がある場合

候補がある場合は、右部品パネルで候補一覧を表示する。

候補行では以下を確認できるようにする。

- 部品名
- 部品種別
- Cal / Ref
- グレード
- 上代
- 仕入
- 在庫数
- 仕入先

候補選択時は、対象の明細行だけに反映する。

反映対象は以下。

- 部品名
- 上代
- 仕入
- 数量
- PartsMasterId
- 部品Ref
- グレード
- 備考
- 在庫数
- 仕入先

## PartsMaster候補がない場合

候補がない場合は、右部品パネルから次の導線を出す。

- Web検索
- 新規PartsMaster登録
- 在庫部品登録
- 手入力で明細挿入

検索結果メモ/詳細エリアは作らず、必要な情報は登録フォームへ直接入力する。

## 閲覧モード

既存案件の閲覧モードでは、右部品パネルで検索・閲覧はできるが、明細反映は行わない。

閲覧モードでの方針。

- PartsMaster候補の閲覧は可能
- Web検索は可能
- 明細への選択反映は不可
- DB保存は発生しない
- 編集が必要な場合は編集モードへ切り替える

## 段階的な小Task案

### Task 018: Web検索UIを右部品パネルへ移設する設計

現在の検索ダイアログの機能を整理し、右部品パネル内に配置するUI単位を決める。

### Task 019: 右部品パネルの表示状態を整理する

未選択、検索中、候補あり、候補なし、閲覧モード、編集モードの表示差分を整理する。

### Task 020: PartsMaster候補選択から明細反映までの仕様を固定する

候補選択時に反映するフィールド、閲覧モードで無効化する操作、別行誤反映防止条件を明文化する。

### Task 021: 候補なし時の新規登録 / 在庫登録導線を設計する

右部品パネルから登録フォームへ渡す初期値と、登録後に明細へ戻す情報を整理する。

## 完了条件

- 右部品パネルの役割が明文化されている
- 明細欄と右部品パネルの責務が分離されている
- Web検索UIを右部品パネル内へ移す方針が記録されている
- 検索結果メモ/詳細エリアを作らない方針が記録されている
- 今後の小Task案が整理されている
- 実装コードを変更していない
